# 📚 시나리오 설계 — NEURAL RUST

> 본 문서는 **StoryManager의 씬 ID와 1:1 동기화**를 기준으로 작성합니다.

---

## 1) 진행 단위

- 기준 단위: Day
- 내부 페이즈: `start` / `noon` / `evening` (확장 가능)
- 실제 코드 연동: `StoryManager.STORY_DATA`

---

## 2) 현재 등록된 씬 ID (코드 기준)

| Day | phase | id | once |
|---|---|---|---|
| 1 | start | `day1_intro` | true |
| 2 | start | `day2_start` | true |

> Day 3 이후는 문서 우선 설계 후 코드 등록.

---

## 3) 씬 작성 템플릿

```md
### Day N - phase
- id: dayN_xxx
- 목적: (세계관 소개 / 인물 갈등 / 시스템 튜토리얼 등)
- 트리거 조건: (기본/플래그/자원)
- 결과 플래그: (setFlag 목록)
- 다음 연결: (다음 씬 id 또는 일반 루프 복귀)
- 대사/연출 메모:
  - ...
```

---

## 4) 단기 스토리 확장 계획 (3~7일차)

1. Day 3: 공방 루프 정착(관리/탐사대 설명)
2. Day 4: 첫 위기 이벤트(부식/자원 압박)
3. Day 5: 세력 단서 노출
4. Day 6: 탐색 선택의 결과 차이 체감
5. Day 7: 중간 결산 + 다음 장(章) 진입

---

## 5) 규칙

- 씬 ID는 소문자 스네이크케이스 사용
- 문서에 ID 작성 후 `StoryManager`에 동일 ID 등록
- once/condition/onComplete 여부를 문서에 반드시 표시
