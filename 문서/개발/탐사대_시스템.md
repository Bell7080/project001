## 1. 탐사대 개요

탐사대는 탐색(ExploreScene)에 파견하는 인원 편성 단위다.
세계관 설정상 탐사는 **심해 잠수정**을 타고 진행된다.
잠수정 내부의 **3×3 구역에 인원을 배치**하여 진입하는 심해 괴수로부터 잠수정을 지키는 디펜스 방식이다.

```
               [상-좌]  [상-중]  [상-우]
[잠수정]  ←  [중-좌]  [중-중]  [중-우]  ←  [괴수 입구]
               [하-좌]  [하-중]  [하-우]
```

---

### 생존 / 사망 시스템

탐사는 **자원과 인원을 소모하며 운영**하는 경영 구조를 가진다.

| 상황 | 결과 |
|------|------|
| 탐사 **성공** (잠수정 생존) | 생존 인원이 전리품(아이템/아크)을 가지고 귀환 |
| 탐사 **실패** (잠수정 파괴) | 탐사에 나간 **전원 사망**, 전리품 없음 |
| 전투 중 **인원 사망** | 해당 인원은 **영구 사망** — 귀환 불가, 교체 필요 |
| 탐사 **귀환** (생존) | 전투 중 받은 피해를 **그대로 유지**한 채 복귀 (건강 스탯으로 회복) |

> 인원을 소모하며 갈아 끼우고 전략을 수정하는 **로그라이크형 운영**이 핵심이다.

## 2. 직업 목록

탐사대에 배치할 수 있는 직업은 총 **3종**이다.

| 직업 | 분류 | 핵심 역할 | 구현 상태 |
|------|------|-----------|-----------|
| 낚시꾼 (Fisher) | 전투 특화 | 심해 괴수 방어 | ✅ 캐릭터 생성·배치 구현 |
| 잠수부 (Diver) | 비전투 특화 | 아이템 수집 | ✅ 캐릭터 생성·배치 구현 |
| AI | 비전투 특화 | 아크(Arc) 수익 창출 | ✅ 캐릭터 생성·배치 구현 |

> 모든 직업은 전투에 참여할 수 있다. 잠수부와 AI는 전투에 특화되지 않았을 뿐이며, 대신 받는 피해 증가 패널티를 기본 특성으로 지닌다.

## 3. 공통 스탯

모든 직업이 공유하는 기본 스탯 체계.

### 스탯 항목

| 스탯 | 설명 | 구현 상태 |
|------|------|-----------|
| **체력** | 캐릭터의 생명력. 1스탯 = 체력 10 | ✅ `stats.hp`, `currentHp`, `maxHp` 구현 |
| **건강** | 탐사 종료 시 소모된 체력을 해당 수치만큼 회복 | ✅ 스탯 저장, 회복 로직은 미구현 |
| **공격** | 괴수에게 입히는 피해량. 스킬 데미지에도 영향을 줌 | ✅ `stats.attack` 구현 |
| **민첩** | 공격 속도에 영향. 1민첩 = 공격속도 +0.1 | ✅ `stats.agility` 구현 |
| **행운** | 회피율과 치명타 확률에 영향. 1행운 = 치명타 확률 +1%, 회피율 +0.1% | ✅ `stats.luck` 구현 |

### 스탯 최솟값 (구현됨)

어떤 디버프나 패널티가 있어도 이 수치 아래로 내려가지 않는다.

| 스탯 | 최솟값 | 코드 기준 |
|------|--------|-----------|
| 체력 | 1 | `mins[0] = 1` |
| 건강 | 0 | `mins[1] = 0` |
| 공격 | 1 | `mins[2] = 1` |
| 민첩 | 5 | `mins[3] = 5` |
| 행운 | 0 | `mins[4] = 0` |

### 치명타 규칙

- 치명타 발동 시 공격력의 **200%** 피해 적용
- 스킬 피해에도 동일하게 적용됨
- 행운으로 인해 치명타 확률이 **100%를 초과**할 경우, 초과 수치 1당 **공격력 +1** 증가

```
예시) 행운 120
  → 치명타 확률 100% (상한)
  → 초과 행운 20 → 공격력 +20 추가
```

> ⬜ 치명타 연산은 전투 시스템 구현 시 적용 예정.

## 4. 직업별 특성

### 4-1. 잠수부 (Diver)

**[기본 특성] 받는 피해 20% 증가** — 전투에 특화되지 않은 직업 패널티.

**탐사 완료 시 생존할 경우 아이템을 획득한다.**

획득 아이템은 고철(스크랩)부터 기록물까지 다양한 종류로 구성된다. 세부 목록은 추후 구체화 예정.

| 가중치 | 결과 |
|--------|------|
| 3 | 아이템 1개 획득 |
| 2 | 아이템 2개 획득 |
| 1 | 아이템 3개 획득 |
| 0.1 | 아이템 4개 획득 |

> ⬜ 아이템 획득 로직은 탐사 결과 처리 시스템 구현 시 적용 예정.

---

### 4-2. AI

**[기본 특성] 받는 피해 20% 증가** — 전투에 특화되지 않은 직업 패널티.

**탐사 참여 시 아크(Arc)를 소모하고, 탐사 완료 시 생존할 경우 아크를 획득한다.**

아크(Arc)는 게임 내 기본 화폐다.

**소모량** — AI의 Cog 등급에 비례하여 1~30 아크 소모 (→ [6. Cog 등급 시스템] 참조)

**획득량**

| 가중치 | 결과 |
|--------|------|
| 3 | 아크 5개 획득 |
| 2 | 아크 15개 획득 |
| 1 | 아크 30개 획득 |
| 0.1 | 아크 50개 획득 |

> ⬜ Arc 소모/획득 로직은 탐사 결과 처리 시스템 구현 시 적용 예정.

---

### 4-3. 낚시꾼 (Fisher)

- 심해 괴물에게 받는 피해 **30% 감소**
- **[특성] 도발** — 괴수의 타겟팅 가중치 산정 시 가중치가 **2**로 적용됨 (다른 직업은 기본 1)
- **[특성] 이동 봉쇄** — 같은 칸의 심해 괴수가 이동 행동을 선택했을 때 발동
  - 75% 확률로 해당 이동 행동을 취소시킴
  - 취소 성공 시 그 행동 턴 동안 괴수를 **경직** 상태로 만듦
  - 별도 방어 수치는 없으며, 스킬이 아닌 **패시브 특성**으로 작동

> ⬜ 직업별 전투 특성은 전투 시스템 구현 시 적용 예정.

## 5. 가중치 확률 시스템

직업 특성의 획득 결과는 **가중치 합산 랜덤** 방식으로 결정된다.

```
전체 가중치 합 = 3 + 2 + 1 + 0.1 = 6.1

아이템 1개 확률 = 3   / 6.1 ≈ 49.2%
아이템 2개 확률 = 2   / 6.1 ≈ 32.8%
아이템 3개 확률 = 1   / 6.1 ≈ 16.4%
아이템 4개 확률 = 0.1 / 6.1 ≈  1.6%
```

이 구조를 도입한 이유는 **아이템 장착 등으로 가중치를 추가할 때 유연하게 확장**할 수 있기 때문이다.

```
예시) 특정 아이템 장착 시 "아이템 4개" 가중치에 +0.5 추가
  → 새로운 전체 합 = 6.6
  → 아이템 4개 확률 = 0.6 / 6.6 ≈ 9.1% (기존 1.6%에서 상승)
```

## 6. Cog 등급 시스템

### 6-1. 등급 산정 (구현됨)

모든 캐릭터는 **스탯 합계**에 따라 Cog 등급을 부여받는다.
`CharacterManager.calcCog(statSum)` 함수로 산정.

```
스탯 합계   →  Cog 등급
  ~10       →  Cog 1
  ~25       →  Cog 2
  ~45       →  Cog 3
  ~65       →  Cog 4
  ~80       →  Cog 5
  ~95       →  Cog 6
  96+       →  Cog 7
```

**AI 아크 소모량**은 Cog 등급에 연동된다.

| Cog 등급 | 소모 아크 (예시) |
|----------|-----------------|
| Cog 1 | 1 |
| Cog 2 | 3 |
| Cog 3 | 6 |
| Cog 4 | 10 |
| Cog 5 | 16 |
| Cog 6 | 23 |
| Cog 7 | 30 |

> ⬜ 소모량 수치는 추후 확정. 전투 시스템 구현 시 연동 예정.

---

### 6-2. Cog 등급별 스킬 & 패시브 부여 (구현됨)

캐릭터는 Cog 등급이 결정될 때 해당 등급의 **스킬 1개**와 **패시브 1개**를 랜덤으로 부여받는다.
`CharacterManager.createCharacter()` 내에서 `PASSIVE_POOL`, `SKILL_POOL` 참조.

```
Cog 3 등급 캐릭터 →  Cog 3 패시브 풀에서 랜덤 1개
                  →  Cog 3 스킬 풀에서 랜덤 1개
```

> 등급이 오를 때 기존 스킬/패시브가 교체되는지, 추가로 누적되는지는 추후 결정.

---

### 패시브 풀 (구현됨)

패시브는 캐릭터의 **기본 공격 및 스킬이 닿는 범위**를 결정한다.

| Cog | 패시브 풀 |
|-----|-----------|
| 1 | 윗칸 타격, 앞칸 타격 |
| 2 | 앞칸 타격, 현재 칸 타격 |
| 3 | 현재 칸 타격, 대각 타격, 윗칸 타격 |
| 4 | 전열 전체 타격, 대각 타격, 앞칸 타격 |
| 5 | 전열 전체 타격, 현재 칸 타격, 후열 타격 |
| 6 | 전/후열 동시 타격, 후열 타격, 전열 전체 타격 |
| 7 | 전체 칸 타격, 전/후열 동시 타격 |

---

| Cog | 스킬 풀 |

## 7. 행동 결정 시스템

괴수와 캐릭터 모두 **가중치 기반 랜덤**으로 행동을 결정한다.

### 심해 괴수
매 행동 턴마다 아래 중 하나를 가중치 추첨으로 선택한다.
- **공격** — 같은 칸 또는 인접한 캐릭터를 공격
- **이동** — 다른 칸으로 이동 (낚시꾼 특성으로 취소될 수 있음)
- **스킬** — 괴수 고유 스킬 사용

> 가중치 수치는 괴수 종류별로 개별 설계. 추후 몬스터 기획 문서에서 관리.

### 캐릭터
- **공격** — 기본 공격
- **스킬** — 직업/캐릭터 고유 스킬

> ⬜ 행동 결정 시스템 전체는 전투 시스템 구현 시 적용 예정.

## 8. 던전 배치 규칙

### 격자 구조 (구현됨)

던전은 심해를 탐사하는 **잠수정** 내부다.
가운데 행 양 끝에 특수 칸이 하나씩 붙어 총 **11칸**이다.
`Tab_Squad_Grid.js`에서 3×3 격자 + 잠수정 칸(인덱스 9) + 입구 화살표로 구현됨.

```
               [상-좌]  [상-중]  [상-우]
[잠수정]  ←  [중-좌]  [중-중]  [중-우]  ←  [괴수 입구]
               [하-좌]  [하-중]  [하-우]
```

| 칸 | 설명 | 구현 상태 |
|----|------|-----------|
| **괴수 입구** | 오른쪽 끝. 심해 괴수가 진입하는 전용 칸. 인원 배치 불가. | ✅ 화살표 시각 요소 구현 |
| **배치 칸 (9칸)** | 탐사 인원을 배치할 수 있는 3×3 구역. | ✅ 드래그 앤 드롭 배치 구현 |
| **잠수정** | 왼쪽 끝. 핵심 보호 대상. 인원 배치 불가. | ✅ 슬롯 인덱스 9로 구현 |

---

### 칸당 배치 인원 (구현됨)

> `CharacterManager.saveSquad()` / `loadSquad()` — 슬롯 10개(3×3 + 잠수정), 각 최대 배열로 저장.
