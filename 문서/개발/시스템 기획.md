> 개발 방향, 장르, 게임 루프, 직군 구조, 자원 구조를 정리한 핵심 시스템 문서.
> 세계관·스토리 설정은 세계관.md / 시나리오.md를 참조.

## 1. 개발 방향 원칙

| 항목 | 결정 |
|------|------|
| 개발 규모 | 1인 개발. 구현 가능한 볼륨을 최우선 기준 |

## 2. 장르 정의

**스토리 중심 관리 시뮬레이션 + 디펜스**

- 로보토미 코퍼레이션을 레퍼런스로 삼되, 게임성 비중은 낮추고 서사 비중을 높인다
- 직접 조작하는 액션 요소는 최소화. 플레이어는 **배치와 판단**으로 게임을 진행
- 매일 반복되는 운영 루틴이 플레이어를 이 세계에 살게 만들고, 그 루틴 사이에 스토리가 끼어든다
- 로그라이크 요소는 현 단계에서 보류

## 3. 핵심 루프

```
[ 아침 — 경영 페이즈 ]
  스토리 씬 / NPC 대화
  ↓
  공방(Atelier) 에서 편성
  └─ AI 전류 채굴 라인 배치
  └─ 낚시꾼 보초 배치
  └─ 다이버 잠수 목표 설정
  └─ 엔지니어 외주 협상 (필요 시)

[ 점심 — 운영 페이즈 ]
  탐색 결과 이벤트 처리
  └─ 괴수 방어 (낚시꾼 디펜스)
  └─ 잠수 결과 수령 (고철 · 로어 · 기타)
  └─ AI 채굴 Arc 수령

[ 저녁 — 결산 페이즈 ]
  Arc · 자원 정산
  스토리 씬 / 캐릭터 대화

[ 다음날 아침 ]
  전날 이벤트 후일담 / 새 대화 이벤트
  → 아침 경영 페이즈로 순환
```

## 4. 공방 (Atelier) UI 구조

플레이어는 인게임 진입 시 공방 화면에 들어온다.

### 화면 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  [상점] [창고] [도감] [회상]   DAY X  |  XXX Arc   [설정]   │
│ ─────────────────────────────────────────────────────────── │
│                                                              │
│                                                              │
│                    [ 탭 패널 영역 ]                           │
│                                                              │
│                                                              │
│  ─────────────────────────────────────────────────────────  │
│       [영입] [관리] [탐사대] [시설] [외주] [드레지]           │
│                                                              │
│                       [ 탐  색 ]                             │
└──────────────────────────────────────────────────────────────┘
```

### 탭 목록

| 탭 | 위치 | 역할 | 구현 상태 |
|----|------|------|-----------|
| 탐색 | 하단 중앙 (기본 탭) | 탐사 구역 선택 및 출발 | ✅ ExploreScene 연결 |
| 영입 | 우측 상단 | 낚시꾼 / 다이버 고용 | 🔲 플레이스홀더 |
| 관리 | 우측 상단 | 캐릭터 카드 그리드·프로필·HP 회복 | ✅ 구현 완료 |
| 탐사대 | 우측 상단 | 탐색에 나갈 팀 편성 | ✅ 구현 완료 |
| 시설 | 우측 상단 | 공방 내실 강화 (탐사대 인원 증가 등) | 🔲 플레이스홀더 |
| 외주 | 우측 상단 | 엔지니어에게 하드웨어 제작 의뢰 | 🔲 플레이스홀더 |
| 드레지 | 우측 상단 | AI를 하드웨어에 호출/삽입 | 🔲 플레이스홀더 |
| 상점 | 좌측 상단 | — | 🔲 플레이스홀더 |
| 창고 | 좌측 상단 | 고철·기록물 등 수집 아이템 관리 | 🔲 플레이스홀더 |
| 도감 | 좌측 상단 | 캐릭터 특성·용어 사전 | 🔲 플레이스홀더 |
| 회상 | 좌측 상단 | 지난 스토리 씬 다시 보기 | 🔲 플레이스홀더 |

## 5. 자원 구조

### Arc (아크) — 핵심 화폐

- 바다에 매립된 전선에서 흘러나오는 전류를 정제한 에너지 단위
- `SaveManager`의 `neural_rust_save` 키에 `arc` 필드로 저장됨
- AtelierHUD에서 현재 Arc 잔액을 실시간 표시

| 지출 항목 | 내용 | 구현 상태 |
|-----------|------|-----------|
| 낚시꾼 고용 | 일당 Arc 지급 | 🔲 영입 탭 미구현 |
| 다이버 고용 | 잠수 의뢰 Arc 지급 | 🔲 영입 탭 미구현 |
| 엔지니어 외주 | 하드웨어 제작 의뢰 Arc + 고철 | 🔲 외주 탭 미구현 |
| 몸체 수리 | 부식된 AI 하드웨어 유지보수 | 🔲 미구현 |
| HP 회복 | 관리 탭에서 캐릭터 회복 (Arc 소모) | ✅ Tab_Manage_Utils._doHeal() |

### 고철 (Scrap)

- 다이버가 심해에서 수집하는 붕괴 이전 문명의 금속 잔해
- 엔지니어에게 Arc와 함께 지급해 AI 하드웨어(몸체) 제작에 사용
- 고철의 시대와 품질에 따라 제작 몸체 성능이 달라짐
- 소금바다 증기에 의해 몸체가 서서히 부식(Rust)되므로 수리용으로도 소비
- ⬜ 고철 자원 저장 구조 미구현 (SaveManager 확장 필요)

### 기타 수집물

- 로어(기록물), 출처 불명 교역품, 세력 해금 트리거가 되는 특수 유물 등
- ⬜ 각 물품 활용 방식 미정

## 7. Cog (코그) — 위험도 등급 시스템

Cog는 이 세계의 **위험 등급 분류 체계**. 탐색 구역, AI, 괴수, 기록물 등에 모두 적용되는 공통 척도.

> 세력을 칭하는 용어: **마스트(Mast)**. 각 마스트의 수장은 **앵커(Anchor)**.

### Cog 등급 체계

| 등급 | 기준 | 구현 상태 |
|------|------|-----------|
| Cog 1 | 스탯 합계 ~10. 일반적인 위험. | ✅ calcCog() |
| Cog 2 | 스탯 합계 ~25. 중급 위험. | ✅ |
| Cog 3 | 스탯 합계 ~45. 고위험. | ✅ |
| Cog 4 | 스탯 합계 ~65. 극위험. | ✅ |
| Cog 5 | 스탯 합계 ~80. | ✅ |
| Cog 6 | 스탯 합계 ~95. | ✅ |
| Cog 7 | 스탯 합계 96+. 최고 등급. | ✅ |

> 탐색 구역·괴수 등 인게임 Cog 적용은 전투 시스템 구현 시 연동 예정.

## 8. 세력 시스템

확정 용어: **마스트(Mast)** — 세력 단위 / **앵커(Anchor)** — 각 마스트의 수장

### 세력 해금 조건

- 일정 Arc 축적 **또는** 일정 일차 경과 시 세력 시스템 해금
- 해금 이후 각 세력과 개별적으로 관계를 쌓아야 소속 인력을 고용 가능

### 세력 고용의 장점

| 구분 | 일반 고용 | 세력 고용 |
|------|-----------|----------|
| 비용 | 낮음 | 높음 (Arc 추가 필요) |
| 성능 | 기본 | 차별화된 특기 보유 |
| 스토리 | 없음 | 해당 세력 관련 이벤트 연동 |
| 리스크 | 낮음 | 세력 내부 정치에 엮일 수 있음 |

> ⬜ 세력 시스템 전체 미구현.

## 9. 부식(Rust) 시스템

- AI 몸체는 소금바람에 의해 매일 조금씩 부식된다
- 부식이 쌓이면: 능력치 저하 → 기능 장애 → 야생화
- 수리에는 Arc + 고철이 소비됨. 고품질 고철일수록 수리 효율 상승
- **설계 의도**: 자원 관리의 핵심 압박 요소
- ⬜ 부식 시스템 전체 미구현.

## 12. 개발 우선순위

```
1단계 — 뼈대 ✅ 완료
  └─ 공방(Atelier) 씬 기본 탭 레이아웃
  └─ 캐릭터 생성·관리·탐사대 배치 시스템
  └─ 설정(폰트·오디오·키·저장) 시스템
  └─ 탐색(ExploreScene) 카드 추첨 UI

2단계 — 서사 ⬜ 예정
  └─ ExploreScene 카드 선택 후 씬 분기
  └─ 드레지(Dredge) 시스템 구현
  └─ 스토리 씬(STORY_DATA) 작성 및 연출
  └─ 로어 해금 연동

3단계 — 핵심 루프 ⬜ 예정
  └─ Arc 수입 루프 (탐사 결과 → Arc 획득)
  └─ 낚시꾼 디펜스 (전투 시스템)
  └─ 다이버 수집 루프 (고철·로어)
  └─ 부식 시스템

4단계 — 세계 ⬜ 예정
  └─ 세력(마스트) 시스템 해금 및 관계 구조
  └─ NPC 대화 시스템
  └─ 영입 탭 구현

5단계 — 확장 ⬜ 예정
  └─ 세력 소속 전문 인력 고용
  └─ 고철 품질 등급
  └─ 추가 AI 캐릭터 스토리
  └─ Electron 패키징 → exe
```
