# ⚙️ 시스템 기획 — NEURAL RUST

> 코드 구현 상태를 반영한 시스템 설계 문서.
> 기획 목표와 현재 구현을 분리해 기록합니다.

---

## 1) 허브 구조 (Atelier)

### 1-1. 현재 구현
- 상단: Day / Arc HUD, 설정/로비 이동 버튼
- 좌측 탭: 상점·창고·도감·회상 (현재 스텁)
- 우측 탭: 영입·관리·탐사대·시설·외주·드레지 (관리/탐사대만 실구현)
- 하단 중앙: 탐색 탭 진입 버튼
- 첫 진입 시: Welcome 오버레이 표시

### 1-2. 목표
- 스텁 탭을 실제 경제/성장 루프와 연결
- Day Phase에 따라 탭 잠금/해금 상태 제공

---

## 2) 탐색 시스템 (Explore)

### 2-1. 현재 구현
- 3 슬롯 스핀 연출
- 전투/스토리 타입 카드 후보 제공
- 사용자가 1개 선택 가능
- 선택 후 현재는 공방으로 복귀

### 2-2. 목표
- 카드 타입/코그 기반 씬 분기
- 결과 보상(Arc/고철/로어/플래그) 반영

---

## 3) 관리 시스템 (Manage)

### 3-1. 현재 구현
- 캐릭터 카드 목록 표시
- 직군/코그/상태 정보 확인
- 프로필 팝업(`Tab_CharProfile`) 연동
- 회복(Arc 소모) 등 유틸 기능 제공

### 3-2. 목표
- 상태이상/부식/피로도 등 장기 상태 관리
- 장비/모듈 슬롯 추가

---

## 4) 탐사대 시스템 (Squad)

### 4-1. 현재 구현
- 3x3 격자 배치
- 슬롯당 최대 3인 배치
- 드래그/드롭 + 회수
- 직군 필터, 슬라이더 기반 캐릭터 선택
- 편성 데이터 `nr_squad` 저장

### 4-2. 목표
- 칸별 포지션 보정(전열/후열/대각)
- 이벤트/전투 결과에 편성 영향 반영

---

## 5) 스토리/진행 시스템

### 5-1. 현재 구현
- `StoryManager` 기반 Day/Phase 데이터 관리
- Day 1/2 시작 씬 ID만 등록
- `TOTAL_DAYS = 50` 임시값

### 5-2. 목표
- Day별 씬 목록 및 조건 분기 확장
- 엔딩 조건, 루트 분기, 회상(메모리) 연동

---

## 6) 저장/설정 시스템

### 현재 구현
- SaveManager: 세이브/스토리/설정 저장
- InputManager: 키 바인딩
- AudioManager: 마스터/BGM/SFX
- FontManager: 폰트 저장
- CharacterManager: 캐릭터/탐사대 저장
